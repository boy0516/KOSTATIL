** 스크럼 프로세스 
-> 스크럼 현황판(보드판)
-> product backlog(기능목록)

1. product backLog 만들기 
 -> user story를 추출
ex) 쇼핑물 프로젝트한다하면 
 회원관리                 제품관리               주문관리  
-회원가입                -제품목록              -주문등록
-회원탈퇴                -제품등록              -주문목록
-회원수정                -제품수정/삭제       - 결제관리
-로그인                                             -배송관리
-아이디/패스워드 
 (찾기)

2. 스프린트 기간과 소유인원 계산 
  - 8(시간) * 5(명) * 2(주) = 80(시간)

3. 각각의 user story에서 각 기능의 예상소유시간 산출 
  -> 전체 팀원이 합의에 의해서 산출 
  -> 카드(1/4, 1/2, 1, 2, 3, 5)

4. 소유시간 내에서 user story에서 가장 우선순위를 선정 
 -> 스프린트 product backlog 추출 

5. 각각의 기능 목록에 대한 포스트잇 만들기(담당자 고유한 색깔)
  - 고유번호(알파벳과 숫자가 포함된)
  - 제목/내용(제목이 길면 좋다, 구체적으로)
  - 소유시간 
  - 담당자  


6. 스프린트 현황판 구성 

product backlog 목록    TODO    DOING(진행중)    DONE(완료)  ISSUE(하면서 이런부분에 문제가 있다, 협력해서 도와달라)


7. 일일 스프린트 회의
  - stand 회의(모두 일어나서 회의참가) -> 소요시간 : 10분 ~ 15분  
  - 스코럼마스터 질문 
   1) 어제 뭐 하셨나요?
   2) 어제 작업 중에 문제는 없었나요?(팀원이 숙지해야하는 문제)
   3) 오늘은 할 예정은 무엇인가요?
    -> 대답할 때 포스트잇을 이동
